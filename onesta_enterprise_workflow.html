⚙️ Operations Function
Fulfillment & Customer Journey

Operations Onesta dirancang zero-touch untuk transaksi standar. Setiap order berjalan melalui pipeline otomatis dari confirmed payment hingga delivered product dan 30-day journey activation.

1
Order Received
Midtrans webhook → Jubelio sync
Payment confirmed di Midtrans memicu webhook ke Onesta Store backend (Laravel). Order otomatis masuk ke Jubelio untuk inventory deduction dan fulfillment queue. Tidak ada input manual.
Midtrans webhookJubelio auto-syncZero manual input
2
Inventory Check
Jubelio — real-time stock management
Jubelio mengelola inventory secara real-time lintas channel. Stok di bawah threshold otomatis trigger reorder alert. Laporan inventory sync ke Accurate GL untuk pencatatan COGS.
JubelioReal-time stockAuto reorder alertCOGS sync
3
Shipping
Biteship — multi-kurir otomatis
Biteship mengelola multi-kurir (JNE, J&T, SiCepat, dll) dengan rate comparison otomatis. AWB generation, pickup scheduling, dan tracking integration ke Onesta Store customer portal.
BiteshipMulti-kurirAuto AWBLive tracking
4
Customer Notification
WooWA — automated status updates
WooWA mengirim notifikasi WA otomatis di setiap milestone: Order Confirmed → Processing → Shipped (+ no resi) → Out for Delivery → Delivered. Customer selalu ter-update tanpa CS intervention.
WooWA5 status milestonesAuto WA notify
5
Delivery Confirmed
Journey activation → D1 onboarding
Status "Delivered" memicu Chatrigo sequence D1: Ritual protocol personal, 3 Golden Rules, invite komunitas, QR traceability story. Ini adalah "moment of delight" pertama setelah produk di tangan.
Delivery triggerChatrigo D1Ritual protocolDelight moment
6
Returns & Exceptions
Exception handling — human escalation
Kasus di luar SOP (komplain, retur, kerusakan) otomatis di-flag ke CS team via dashboard Onesta Store. SLA response maksimal 4 jam. Setiap kasus direkam untuk quality improvement.
Exception flaggingCS dashboard4-hour SLA

💻 IT Function
Platform, Security & Reliability

IT Onesta dibangun di atas arsitektur yang sudah production-grade. Fokus IT adalah memastikan uptime, integrasi antar sistem berjalan mulus, dan data customer terlindungi.

1
Core Platform
Onesta Store — Next.js 15 + Laravel 10
Frontend Next.js 15 (onesta.co.id) + Backend Laravel 10 (API + webhook handler). MySQL 8 database. Alma Linux 9.4 OS, Apache 2.4, 8 Cores CPU, 16GB RAM, 400GB SSD. Dev environment terpisah di Vercel.
Next.js 15Laravel 10MySQL 8Alma Linux
2
Integration Hub
Webhook orchestration — semua third-party
Laravel backend sebagai integration hub: menerima webhook dari Midtrans → forward ke Jubelio → trigger WooWA → update GA4 events. Setiap integrasi punya error handling dan retry logic.
Webhook hubError handlingRetry logicEvent log
3
Diagnostic Tools
Balance Check™ + Bot Hub — GitHub Pages
Semua diagnostic tools (Balance Check V2, Growth Flywheel, Bot Hub) di-host di GitHub Pages sebagai static files — zero server cost, globally distributed CDN, 99.9% uptime otomatis.
GitHub PagesStatic hostingZero server costCDN global
4
Reporting System
Onesta Report — React + Laravel 12
report.onesta.co.id — internal dashboard untuk manajemen. Data dari Jubelio (inventory + order) disajikan dalam laporan real-time. Finance, ops, dan management bisa akses data dari satu tempat.
report.onesta.co.idReact + Laravel 12Jubelio data source
5
Security
HTTPS + data protection layer
Semua komunikasi via HTTPS. Payment data tidak menyentuh server Onesta (Midtrans PCI-DSS compliant). Customer data di MySQL dengan encryption at rest. API keys di environment variables, tidak di code.
HTTPS enforcedPCI-DSS via MidtransData encryptionEnv variables
6
Future Roadmap
Balance Score API — data moat Onesta
Jangka menengah: Balance Check hasil diagnostik tersimpan di database per user → tracking score trend → personalisasi semakin dalam → data ini menjadi competitive moat yang tidak bisa ditiru.

💰 Finance Function
Revenue, COGS & Reporting

Finance Onesta didesain untuk automated reconciliation — setiap transaksi dari Midtrans mengalir otomatis ke Accurate GL, memberikan P&L visibility real-time tanpa manual entry.

1
Revenue Recognition
Midtrans → Accurate GL auto-journal
Setiap payment confirmed di Midtrans otomatis membuat jurnal penjualan di Accurate GL. Revenue diakui pada saat delivery confirmed (bukan payment) sesuai standar akuntansi. Tidak ada manual entry.
MidtransAccurate GLAuto-journalAccrual basis
2
COGS
Jubelio → COGS per SKU → Accurate GL
Jubelio mencatat setiap unit keluar dengan HPP (Harga Pokok Penjualan) per SKU. Data ini sync ke Accurate GL untuk COGS calculation otomatis. Gross margin visible per produk, per channel, per periode.
Jubelio COGSPer-SKU marginAuto sync
3
Opex Tracking
Tool subscription cost management
Semua subscription tools (Chatrigo, WooWA, Biteship, Jubelio, dll) dicatat sebagai opex di Accurate GL dengan kategorisasi per fungsi. Monthly SaaS spend dashboard untuk cost control.
SaaS opex trackingCost by functionAccurate GL
4
Reporting
Onesta Report + Accurate GL — P&L real-time
Management dashboard di report.onesta.co.id menampilkan: Revenue, COGS, Gross Margin, Opex, EBITDA per periode. Data dari Jubelio + Accurate GL disajikan dalam satu view untuk keputusan yang cepat.
P&L dashboardReal-timePer channel view
5
Cash Flow
Subscription MRR model — predictable revenue
Model subscription (1/3/6 bulan) menciptakan MRR yang predictable untuk cash flow planning. Accurate GL tracking: AR aging, payment terms per paket, dan subscription renewal forecast.
MRR trackingSubscription modelAR agingCash forecast
6
Compliance
Tax, audit trail & financial control
Accurate GL menyimpan audit trail lengkap untuk setiap transaksi. PPN tracking otomatis dari Midtrans. Laporan keuangan siap audit dengan dokumen source dari Midtrans + Jubelio sebagai pendukung.

🛡️ Risk & Control
Enterprise Risk Register

Light-asset model mengurangi banyak risiko operasional — tapi menciptakan risiko baru di area integrasi, data, dan platform dependency. Ini adalah risk register yang harus dipantau.

AreaRiskLevelControl
Marketing Platform dependency — jika WA policy berubah, broadcast terdampak Medium Diversifikasi channel: email (Zoho), IG, web organic sebagai backup
Sales Bot Chatrigo down → conversion terhenti High Fallback ke WA Business manual + CS team. SLA bot recovery <2 jam
Sales Midtrans outage → payment failure High Monitor Midtrans status page. Alternatif: manual transfer sementara + CS guide
Operations Jubelio sync failure → oversell / stock discrepancy High Daily inventory reconciliation manual. Buffer stock minimum 10% di atas forecast
Operations Biteship API down → shipping delay Medium Fallback ke booking kurir manual. Customer WA proactive notification
IT Server down → onesta.co.id tidak bisa diakses High Monitoring 24/7. Dev environment di Vercel sebagai hot standby. Backup harian
IT Data breach — customer data exposed Critical HTTPS enforced, data encryption, API keys di env vars, regular security audit
Finance Accurate GL sync failure → incorrect P&L Medium Weekly manual reconciliation. Midtrans settlement report sebagai source of truth
Retention Chatrigo sequence error → wrong message ke wrong user Medium Test sequence sebelum deploy. Monitoring first 24 jam setiap campaign baru
Platform Key tool price increase → opex blow-up Low Annual contract review. Identify alternative tools per kategori sebagai backup vendor
Control Framework
D
Daily Controls
Operasional monitoring harian
Cek: Midtrans settlement vs Accurate GL · Jubelio stock level · Chatrigo sequence aktif · WooWA delivery rate · Server uptime status.
W
Weekly Controls
Business performance review
Review: CVR diagnostic → order · D30 completion rate · Gross margin per SKU · Churn signals · NPS score · Opex vs budget.
M
Monthly Controls
Strategic & financial review
Review: P&L full · MRR growth · LTV:CAC ratio · Tool stack ROI · Risk register update · Roadmap progress · Onesta Buddies performance.
Light-Asset Advantage for Risk: Karena Onesta tidak punya warehouse sendiri, tidak punya delivery fleet, dan tidak punya server on-premise — sebagian besar operational risk di-transfer ke vendor (Jubelio, Biteship, hosting provider). Fokus risk management adalah pada integration reliability dan data integrity.
